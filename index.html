Io有丰富的并法库，其主要组成部分包括协程、actor、future。

协程

协程是并发的基础，提供了进程的自动挂起和恢复执行的机制。每次yield都会自动挂起当前进程，
并把控制权转移到另一进程当中。通过在消息前加@或@@可以已补触发消息，前者返回future，后者返回nil。

actor

一个actor可以改变其自身的状态，并且通过严格控制的队列接触其他actor。

发送异步消息给对象就是actor。

以上代码可以通过io filename.io执行或者通过Io交互解释器执行。

io filename.io

Io交互解释器

上述四行定义部分一致，不再赘述。

通过执行slow @@start ； fast @@start,会立即返回nil。

注意

  • 时间变化，在交互解释器中为光标闪烁
  • 注意有无wait()的区别，wait()代表等待时间，slower @@start;faster @@start;wait(3)，使线程在系统终止前执行完毕。
    我们同时运行了两个线程，只不过发送异步消息给两个对象，它们就变成了actor。

    future

    future是在异步消息调用时立即返回的一个结果对象。由于被调用的消息可能需要一段时间进行处理，因此到最终产生结果的时候，future就会
    变成这个结果值。如果在结果尚未产生时请求future的值，进程就会阻塞直到产生结果为止。

哎，这次做free code camp的编程作业，简直太苦逼了，没有想到会出现这么
多问题。

编程要求说明

  1. 一个按钮负责转换引语，按下之后它会给出另外一个随机引语
  2. 另外一个按钮可以发送一个推文向其他人分享

任务流程

页面文档

<body id="body">
<h1>my quotes</h1>

</div>

转换按钮

  1. 创建一个引语列表，比如是一个含有十个引语字符串的列表quote=[....]；
  2. 随机函数var index = Math.floor(Math.random() * (quote.length));
  3. 向页面添加引语 document.getElementById("quotes").innerHTML = quote[index];
window.onload=function(){

var index = Math.floor(Math.random() * (quote.length));
document.getElementById("quotes").innerHTML = quote[index];

};

function another() {
var two = Math.floor(Math.random() * (quote.length));
document.getElementById("quotes").innerHTML = quote[two];



}
